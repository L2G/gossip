Installation

  1. gem install gossip
  2. Set up site-wide defaults.
  3. Copy whichever of the scripts you want into somewhere in your $PATH.

Details

  1. Install the gem in the normal way. (Gossip has dependencies on a number of other gems. You may be annoyed, therefore, when gem installs the Twitter gem even though you aren't going use Twitter. If you're annoyed, send me mail, and I'll make a configurable release.)

  2. Gossip can send messages to Twitter, email, Jabber, etc. Each of those destinations will require some information (username, password, etc.). All that can be specified on the command line, in configuration files, or it can be set as site-wide defaults.

    Gossip comes with a set of site-wide defaults that will certainly be wrong for you. They are in lib/gossip/site-config.rb. You have some choices.

    • You can leave them alone and make the user specify them in a single config file that applies to all Gossip commands. That makes sense, for example, if you'll be using those commands on a shared server. If everything you do is done under a single user account, putting a .gossiprc file in that account's home directory is easy.

    • You can edit site-config.rb to change the values. Note, however, that you'll lose the values if you install a new version.

    • You can put an improved site-config.rb somewhere else. In that case, you'll have to change the require "gossip/site-config" line in each of the Gossip scripts before installing them.

    To modify site-config.rb, see the instructions within it. The explanation of the system metaphor might help understand the jargon.

    To make a user configuration file, see Using Bundled Scripts.

  3. Gossip comes with some scripts. They're not automatically installed for fear of conflicts with existing commands. Instead, install the ones you want manually. Here's the current set:

    • fanout sends a message to multiple destinations.
    • svntell constructs a message from a Subversion repository and sends it. It's to be called from a subversion post-commit hook.
    • watchdog runs a command line, then reports on its output and duration.

    See Using Bundled Scripts for more.

    Try a script before installing them. A good thing to try is ./fanout --help. If you get a complaint that Ruby's not found, you'll need to change the "shebang" line (the first line in the script that now says #!/usr/bin/env ruby) to the absolute path name of your version of Ruby.
